button

A cform element using images to act as button.

  • Note: Use only on a 'cform', not a 'form'
  • Instantiating cform elements without using "cform" is not recommended.
Syntax
button, {param}
Creates a button.
Parameters are given as {param:value, param:value, ...}
Parameter Summary
Attributes Parameter Name and Description
 
name:"button_text"
This sets the text on the button, which is also the button id.
Update: If the name contains the string "toggle" (e.g. toggle1, toggle_option, etc.), the button acts in toggle mode, where the "clicked" state is held. For best results, make the "click" image different from "base" image.
 
x:offset_from_top_left
Horizontal position of the button from the top-left of viewport window.
 
y:offset_from_top_left
Vertical position of the button from the top-left of viewport window.
Optional
w:width
Width of button. If base images are provided, this parameter is ignored.
Optional
h:height
Height of button. If base images are provided, this parameter is ignored.
 
base:"path_to_base_image_of_button"
base:"valid_HTML_color"
This is the default image or color of the button.
Valid HTML colors may be specified as "string" (e.g. "red"), "hex" (e.g. "#RRGGBB"), "rgb(rr,gg,bb)" or "rgba(rr,gg,bb,aa)".
Optional
hover:"path_to_hover_image_of_button"
hover:"valid_HTML_color"
This is the hover image of the button. If not given, the base image is used.
Valid HTML colors may be specified as "string" (e.g. "red"), "hex" (e.g. "#RRGGBB"), "rgb(rr,gg,bb)" or "rgba(rr,gg,bb,aa)".
Optional
click:"path_to_click_image_of_button"
click:"valid_HTML_color"
This is the image of the button when clicked. If not given, the base image is used.
Valid HTML colors may be specified as "string" (e.g. "red"), "hex" (e.g. "#RRGGBB"), "rgb(rr,gg,bb)" or "rgba(rr,gg,bb,aa)".
Optional
showText:boolean (true/false)
Shows text at button center if true. No text if false. Default is true.
Optional
link:[keyword-value pair]
The value of link is executed if the button is clicked.
The keyword-value is exactly as you would declare it in a scriptline but enclosed in square brackets ([...]).
If link is not given, nothing happens on a user click.
Note: Technically, any keyword-value is allowed, but use only those keywords needed for flow control, such as "jump" and "set". "message" can also be used.
Optional
tip:"tool_tip_to_display"
The tooltip to display when mouse hovers over the button.

Example

This creates an "Options" button at location [360, 400], uses the given images depending on the button state, and displays a message when clicked. 
  
chapter = [

	...
	button, {name:"Options", x:360, y:400, 
		base:"base.png", 
		hover:"hover.png", 
		click:"click.png", 
		link:[message, "Not yet implemented!"]},
	...
];
