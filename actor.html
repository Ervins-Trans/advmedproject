actor

Creates an actor and/or display dialog in script box.

Syntax
actor, {param}
Creates an actor with one or more sprites and displays both the character and dialog on screen.
Actors are bottom-center aligned and are auto-positioned on the screen by default.
Parameters are given as {param:value, param:value, ...}
"actor_id", {params...}
Update: Shortcut for
actor, {id:"actor_id", params...}
where "actor_id" is the id/name of the actor. The actor must have been previously defined before using this shortcut.
"actor_id", "some_text"
Update: Shortcut for
actor, {id:"actor_id", say:"some_text"}
where "actor_id" is the id/name of the actor and "some_text" is the dialog to say. The actor must have been previously defined before using this shortcut.
Parameter Summary
Attributes Parameter Name and Description
 
id:"actor_id"
A unique identifier for the actor.
 
sprite:["tag", "path_to_sprite_file", "vertical_alignment"]
sprite:["tag", "path_to_sprite_file", "vertical_alignment", frames, fps, repetitions]
sprite:"tag"
Tag is the identifier of the sprite, sprite file is the filename of the image file to use as actor sprite.
Actors can have multiple sprites, but declare each one at a time as needed, to minimize load time. Each declared sprite for the actor are identifiable using "tag".
Update: Once declared, the sprite can be accessed just by specifying the "tag". For example, when reusing the sprite.
Update: Now accepts a user variable as path to sprite file.
Update: Now accepts an optional third parameter specifying the vertical alignment of the sprite. Valid values are "floor" (default if not given), "bottom", "center", "top" and "roof".
FLOOR and ROOF offsets the sprite according to actorYPosition in Config file.
BOTTOM and TOP aligns the sprite to the corresponding edge of the viewport.
CENTER puts the sprite in center.
Update: Now supports animated sprites. Parameters should be given as
["tag", "path_to_sprite_file", "vertical_alignment", frames, fps, repetitions]
where frames is the number of frames/sprites in a horizontal filmstrip defined by path_to_sprite_file,
fps is the animation frames per second, and
repetitions is the number of animation loops (default is -1 or infinite repetitions).
Note that vertical_alignment, frames, fps and repetitions are all optional. See also Sprites demo.
Optional, Persistent
avatar:["tag, "path_to_avatar_file"]
avatar:"tag"
avatar:"path_to_avatar_file"
avatar:["tag, "path_to_avatar_file", frames, fps, repetitions]
Display a small image of actor in script box.
Update: Displaying avatars need to be enabled in configuration file (on by default).
Update: Now accepts a user variable as path to avatar file.
Update: Now accepts a tag name, just as in sprites. To use a previously declared avatar, use the tag name only.
Update: For compatibility, specifying avatars as an image file without tag is retained. These avatars however can not be accessed by tags.
Update: Now accepts a multi-frame filmstrip image to create an animated avatar, similar to sprites.
Optional
nick:"actor_nickname"
Name to be displayed in script box. If nick is not provided, the id is used.
Update: Now accepts a user variable as nick, for use when player provides a name.
Optional, Actor creation only.
color:"nickname_color"
The color of the nickname when displayed.
Optional
position:"position_of_actor"
Sets the position of the actor, default is "auto".
"auto" - auto-positions actor on viewport
"left" - positions actor on left side of viewport
"right" - positions actor on right side of viewport
"center" - positions actor on middle side of viewport
"back" - positions actor behind other actors
"front" - positions actor in front of other actors
"[left | right | center] [back | front]" - combination of options
Deprecated: See 'z_order'.
Optional, Persistent
z_order:number
This sets the Z-order of the actors. By default, all actors are given a Z-order of 0, and are drawn in the order they were created. A negative Z-order value puts the actor behind, while a positive Z-order puts the actor in front. For best effects, use integer Z-order from -9 to 9.
Note: Z-order is also used when actorPerspective is set in Config file. A value of -10 and below makes the actor move with or slower than the background. A value of 10 and above is not supported.
Optional
show:boolean (true/false)
Displays or hides actor sprite. Upon creation or adding new sprite, it is automatically set to true.
Optional, Persistent
effect:"effect_name [args]"
Update: Effect when showing or hiding sprites.
Supported effects:
- "none" (default)
- "dissolve"
- "fade"
- "ghost" (a semi-transparent graphic)
- "left" | "right" | "top" | "bottom" (slides the actor graphic to and from the specified direction)
- "scale scale_value" (scales the actor graphic relative to base size)
- "rotate rotate_value" (rotate the actor graphic in degrees clockwise relative to previous orientation)
- "translate x_value y_value" (translate the actor graphic according to the given values relative to current position; applies only if actor is not "auto" positioned)
Update: The 'scale', 'rotate' and 'translate' arguments now accept a user variable id.
Update: A 'nowait' argument, e.g. "fade nowait", can now be added to continue without waiting for effects to complete.
Optional, Persistent
time:number
Duration of effect.
Optional
say:"text_to_display"
balloon:"text_to_display"
Text to be displayed in script box. This automatically adds the name tag to the box (like "text" with "speaker" parameter). Script box and text settings (such as fonts) are carried over here.
Update: If "value" is a user variable id, the contents of the user variable will be displayed.
Update: To display user variables within the text, enclose it with '#', e.g. #var_name#.
Update: 'balloon' displays the text in a speech balloon. Name tag and avatar are not displayed in a speech balloon. The fill and stroke style, as well as the size of the speech balloon, can be set in Config file.
Optional, Persistent
append:boolean (true/false)
Appends text in same window. (same as append in "text")
Optional
remove:"sprite_tag" or "actor"
If a sprite tag is given, the sprite is removed from the list of actor sprites. If keyword "actor" is given, the actor is removed from the list of characters.
If the actor or sprite being removed is currently displayed, actor or sprite is hidden automatically.
An actor should have at least one sprite, so its last sprite can not be removed.
Optional
voice:"path_to_sound_file_no_extension"
Voice dub if any. Only if 'say' is not empty

Example

Actor1 is displayed on the stage as given below. 
  
chapter = [

	...
	/* Create and display actor1 without saying anything but set with a red name tag */
	actor, {id:"actor1", nick:"Me", color:"#ff0000", sprite:["normal", "images/actor1-normal.png"], effect:"dissolve"},
	...
	/* Let actor1 say something which automatically opens script box */
	actor, {id:"actor1", say:"Hello world!"},
	...
	/* Hides actor1 */
	actor, {id:"actor1", show:false},
	...
	/* Add a new sprite to actor1 and automatically display actor.
	   The text lines are appended because of append. */
	actor, {id:"actor1", sprite:["smile", "images/actor1-smile.png"], say:"Hello again!"},
	actor, {id:"actor1", say:"And again!", append:true},
	...
	/* Use actor shortcut to display a dialog */
	"actor1", "Goodbye!",
	...
	/* The sprite with tag "normal" is removed */
	actor, {id:"actor1", remove:"normal"},
	...
	/* Actor1 is permanently removed off the stage,
	   Note the "actor" after remove is a predefined keyword for removing the whole character */
	actor, {id:"actor1", remove:"actor"};
	...
];
